git知识

git配置

当你安装Git后首先要做的事情是设置你的用户名称和e-mail地址。这是非常重要的，因为每次Git提交都会使用该信息。它被永远的嵌入到了你的提交中：

　$ git config –global user.name “John Doe” //global表示这个电脑里这个用户的操作范围

　$ git config –global user.email “johndoe@example.com“ //–system表示电脑里所有用户的操作范围

只需要配置一次，以后就再也不用去动它了。如果你传递了 –global 选项，因为Git将总是会使用该信息来处理你在系统中所做的一切操作。如果你希望在一个特定的项目中使用不同的名称或e-mail地址，你可以在该项目中运行该命令而不要–global选项。

查看配置和修改

需要git版本记录时，文件中git init。

git clone之后，文件中会自动有一个.git的隐藏文件，这里存储着文件的以前版本记录，不需要git init。

添加操作

  1. git add test.txt 从工作区推到暂存区
  2. git commit -m “解释和备注” 推到版本库里，只有操作commit才会记录到版本里，只有加-m，后面才能加解释。
  3. push 推到远程仓库

删除操作

  1. git rm test.txt//或者手动删除
  2. commit -m “备注”

撤销修改

  1. 文件在工作区，git checkout – file//注意–之后有一个空格,其实是用版本库里的版本替换工作区的版本

  2. 文件被推到暂存区了，git reset head file

  3. 文件在本地版本库了，用git reset –hard head^^

返回以前的版本

git log查看提交的历史状态

git reflog//记录每一次的命令，可以看以前的命令的记录

git reset –hard HEAD^^//回到上上一个版本，此版本之后的版本不见了

git reset –hard HEAD~10//回到往上10个版本

git reset –hard commit_id//回到任意一个版本，注意用reflog知道版本id，

分支操作

git branch -b newFun//相当于下面两步

git branch newFun//创建分支，也可以在github里创建分支然后git pull 再用git checkout 到这个分支上，

git checkout newFun//切换到这个分支

git branch //查看分支

git branch -d newFun//删除分支

git merge newFun//注意先切换到master再用这个命令，当前分支newFun合并到当前分支master，合并时有冲突，两个文件中有重名的文件且内容不一样，git不知道要保留哪一个

git diff//提示说文件被修改过，查看两个文件的不同

git log –decorate –all –oneline –graph//查看图形化的分支

更新远程代码到仓库，git pull和git fetch的区别。

git pull相当于git fetch和git merge两个操作，拉取新版本之后再与原来的合并。

git fetch更安全也更符合实际要求，因为可以在merge前，我们可以查看更新情况，根据实际情况再决定是否合并。

git remote -v//显示了可以抓取和推送的origin的地址。如果没有推送权限，就看不到push的地址。

git status查看文件是否被修改及工作状态，word及视频图片等二进制存储方式无法被跟踪。

git mv test.txt file.txt//改名字

注意:git checkout branch和git checkout –file,切换到文件要加–,一个-代表简写，两个-代表全写，如-b,–delete。git不区分大小写。
